Variables

RECAP: What is a Variable?

Variables are essential to DRY programming. In the simplest of terms they are containers that hold a value. The variable label is known as a symbol name and provides a human-readable way to identify the container. In reality variables reference locations in memory where the value is stored. However, instead of having to remember the memory address you get to use a simple human-readable label.

 Example:
Address/Label Operator Value
0x12DF = "Morty"
name = "Morty"

RECAP: Data Types

Data is essential to applications. In order to perform normalization, security, and sanitization, programming languages provide some native data types. The following are the most common native data types:

  • Char/String
  • Integer/Float/Double
  • Boolean
  • Array
  • Object

Strict languages like C++ and Java require you to explicitly declare the data type when you instantiate your variable. However, JavaScript is a loosey goosey, duck typed language. This means data types are implied.

Remember our first line of code? 

              
                // Step 1: Create a variable with the symbol name
                // 'greeting' and assign it the value of 'Hello World'
                greeting = "Hello World";

Our variable greeting doesn't have an explicit data type declared. So how does the JavaScript interpreter know our data is a string? It knows because of the duck type rule; it looks like string; it behaves like a string; it must be a string.

 Data Types:
Data Type Example
Number x = 5;
y = 2.5;
String name = "Bob";
char = "a";
Boolean cont = false;
accept = true;
Array myArr = [1, "Bob", 2.3, false];
Object Person = {name: "Shaun", age: 39};

The data type Number encompasses all numbers whether they be integers, floats, or doubles.

JavaScript doesn't have a char data type like many other languages do. However, in some midlevel languages like C++ the String data type is actually a char array. Curiously, many string methods in JavaScript operate the same as the array methods...hmmm...are you pickin' up what I'm puttin' down?

Variable Scope

Variable scope is the visibility of a variable to the application. !!!DON'T CONFUSE THESE WITH CLASS VISIBILIES!!! In JavaScript we can declare a variable's visibility at the time of instantiation. The 3 visibility scopes available are global, functional, and block. We have 3 scope modifier keywords we can use to declare our scope and mutability. They are var, let, and const.

 Scope and Mutability:
Scope Scope Keyword Mutable Example
Global N/A x = 5;
Functional var var name = "Louis";
Block let let age = 40.5;
Block const const dob = "1978-12-22";

Variable Scopes seem to confuse students new to JavaScript the most. The above table will definitely help better understand how they work.

Variables in JavaScript can be instantiated without assigning a value. This means you can declare the variable and give it a value later.

 Example: 
              
                // Instantiated variables x and w
                let x;
                let w;

                // Assigned values to x and w
                x = "Foo";
                w = "Bar";

This can be especially handy if you're waiting for values from an external source like a user or an API.

RECAP: Comparison Operators

Comparison operators allow us to test 2 values against each other. Comparison operators compare at a binary level, which means you're not restricted to just comparing numbers. For example "b" > "a" is true. The following operators are the most common comparison operators you will encounter as a programmer in all programming languages.

Logical operators allow us to evaluate 2 or more conditional expressions. && evaluates to true when both expressions are true. || evaluates to true if either of the expressions are true.

 Comparison & Logical Operators:

Is not equal to != Loosely compares two values are not equal (ignores data type) 5 != "6"; 5 != "5";
Is strictly not equal to !== Strictly compares two values are not equal (evaluates data type) 5 !== "5"; 5 !== 5;

If/Else/Else If Conditions

Conditional control structures, like if conditions, provide pathways through our logic based upon an evaluation. These are crucial to programming as they help us to make our applications personal to the user, more efficient by bypassing unnecessary logic, or multifaceted.

 Comparison Operators:
Command Description Example
if () {} The if statement is the most common in programming. It allows us to control whether a block of logic can be executed or not. if (true) { x = 5; }
else if () {} The else if statement can be chained onto a preceding if block. It provides an alternative set of logic to execute if the previous condition check fails and its condition check passes. else if (true) { y = 6; }
else {} The else statement can be chained onto a preceding if block. It provides an alternative set of logic to execute if all of the other connected condition checks fail. else { y = 6; }

Switch

A switch statement evaluates an expression then performs an operation based on the expression's value. Switch statements are very cool conditional structures. Because the switch argument is what is evaluated, and it can be a variable, expression, or literal value, we can perform some very interesting logic that separates the switch from the standard if structures.

Switch statements are made up of various pieces of syntax that make up the structure. The core component is the switch() command. The parenthesis following encompass our expression to use for evaluation. The code block below explains each syntax component in detail.

There are several ideas that switch statements are inefficient and you should always use if statements instead. The new V8 JavaScript engine performs optimizations based on either using switch or if statements. This means you should always use whatever structure provides the best code readability.  Example: 

// Our variable
let service = "good";
let tip;

// Our switch
switch (service) {
  case "terrible":
    tip = 0.00;
    break;
  case "bad":
    tip = 0.05;
    break;
  case "meh":
    tip = 0.10;
    break;
  case "good":
    tip = 0.15;
    break;
  default:
    tip = 0.20;
    break;
}

Ternary

We can't talk about conditional logic structures and not talk about the most beautiful shorthand in the world! I'm, of course, talking about ternary statements. Ternaries are a quick one-liner syntax that will evaluate a condition and return a value for true or a value for false. Ternaries are great for toggle logic and creating clean program readability.

 Example (hover over red text to see what it does): 
                
let age = 40;
let stmnt = (age >= 40) ? "old" : "young";

In addition to basic ternaries you can also do complex nested ternaries. Though this greatly diminishes readability.
